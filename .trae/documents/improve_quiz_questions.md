# 优化答题系统题库计划

该计划旨在解决答题系统中后期题目重复率高、答案逻辑一致以及覆盖面不足的问题。我们将重新编写题库，使其更具专业性、广泛性和实用性。

## 1. 现状分析
- **问题 A**：从第 29 题开始，大量题目使用了相同的选项（“更加智能的自动化流程”等）和相似的结构，缺乏实际考核价值。
- **问题 B**：部分多选题的答案设置不合理，缺乏区分度。
- **问题 C**：题目覆盖面虽然包含了一些核心工具，但深度不足，没有体现出 2026 年 AI 编辑器的高级特性（如复杂的 Agent 工作流、MCP 插件开发、多模型协同等）。

## 2. 优化目标
- **去重与多样化**：确保每一道题都有独特的选项和针对性的解析。
- **扩展覆盖面**：
    - **工具篇**：深入考核 Cursor (Composer/Index), Trae (Builder/Context), Windsurf (Flow/Cascade), Google Antigravity (Mission Control)。
    - **协议与技术**：MCP (Model Context Protocol) 的具体实现、Skills 调用逻辑、Memory/Engram 的持久化机制。
    - **实战工作流**：Plan 模式与 Spec 模式的区别、Vibe Coding 的最佳实践、自动化测试集成。
- **提升质量**：选项编写符合“干扰项合理”原则，解析部分需具备教育意义。

## 3. 实施步骤

### 第一阶段：题库重新分类与规划
- 将题库划分为四大模块：
    1. **基础工具对比** (1-20题)
    2. **核心技术架构 (MCP/Memory/Skills)** (21-45题)
    3. **现代 AI 开发工作流 (Plan/Spec/Agent)** (46-70题)
    4. **大模型特性与前瞻 (Claude/GPT/DeepSeek)** (71-85+题)

### 第二阶段：编写高质量题目
- **修正多选题**：确保多选题的 `answer` 数组包含多个正确的索引，且选项之间有逻辑关联。
- **引入场景题**：增加“在 XX 情况下，应该使用哪个模式”或“如何通过提示词优化 XX 结果”的题目。
- **更新前沿知识**：加入 2026 年预期的技术演进，如 Claude 4.6 的原生多模态代码理解能力。

### 第三阶段：文件更新与验证
- 更新 [questions.json](file:///e:/develop-lcb/workspace/aipm/src/data/questions.json) 文件。
- 检查 `id` 的连续性和 `type` (single/multiple/boolean) 的正确性。
- 在本地运行 `npm run dev` 验证题目显示是否正常，解析是否对应。

## 4. 预期产出
- 一个包含约 80-100 道高质量、无重复、覆盖面广的 AI 编程评估题库。
- 修复所有答案重复和逻辑错误的问题。
